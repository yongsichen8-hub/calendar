## 4. 核心业务逻辑

### 4.1 预算汇总计算

```
管理对象总预算 = Σ(各分类预算额度)  // 单位：CNY
管理对象已用金额(CNY) = Σ(各分类已用金额CNY)
各分类已用金额(CNY) = Σ(该分类下所有费用 × 汇率转换为CNY)
  - 差旅分类：酒店费用按标准上限封顶（超额部分不计入预算占用）
剩余预算 = 总预算 - 已用金额(CNY)
使用百分比 = 已用金额(CNY) / 总预算 × 100%
```

### 4.2 酒店超额计算

```
对每条酒店费用：
  1. 确定酒店标准上限（根据城市等级 + 是否旺季 + 差旅政策）
  2. 计算间夜数 = (退房日期 - 入住日期) 天数
  3. 最大允许金额 = 间夜数 × 标准上限
  4. 将实际金额转换为标准币种进行比较
  5. 超额 = max(0, 实际金额 - 最大允许金额)
  6. 预算占用 = min(实际金额, 最大允许金额)

出差单超额总计 = Σ(该出差单下所有酒店费用超额)  // 统一转为CNY
年度超额总计 = Σ(当年所有出差单超额)
```

### 4.3 补贴自动生成

```
创建/更新出差单时：
  1. 删除该出差单下所有旧补贴记录
  2. 计算天数 = (返程日期 - 去程日期 + 1)
  3. 境内：
     - 补贴金额 = (天数 - 1) × 每日补贴标准  // 首尾各半天
     - 币种 = CNY
  4. 境外：
     - 确定目的地国家的补贴等级和币种（EUR/USD）
     - 补贴金额 = (天数 - 1) × 海外补贴标准
     - 币种 = 目的地对应币种
  5. 生成补贴费用记录（日期=去程日期，paid=false）
```

### 4.4 城市等级体系

**境内（4级）**：
- 一级：北京、上海、广州、深圳
- 二级：杭州、南京、成都、武汉、重庆、西安、苏州、天津、长沙、郑州、东莞、青岛、昆明、宁波、合肥
- 三级：大连、厦门、珠海、无锡、佛山、济南、福州、温州、泉州、南宁、贵阳、海口、三亚、拉萨
- 四级：其他所有城市

**境外（7级）**：
- 根据国家/地区分级，每级对应不同的酒店标准和补贴标准
- 欧洲国家使用欧元（EUR），其他地区使用美元（USD）
- 城市→国家映射表内置于 travelPolicyService

### 4.5 旺季判定

- 旺季月份：7月、8月、9月、10月
- 仅影响境内一级城市和二级城市的酒店标准

### 4.6 币种转换

```
toCNY(amount, currency):
  CNY → amount
  EUR → amount × exchangeRate
  USD → amount × exchangeRate  // 近似处理

fromCNY(amount, currency):
  CNY → amount
  EUR → amount / exchangeRate
  USD → amount / exchangeRate
```

## 5. 前端组件设计

### 5.1 页面结构

```
LedgerPage
├── SubjectListView（管理对象列表页）
│   ├── SubjectCard × N（管理对象卡片）
│   └── 新建/编辑表单（内联）
└── SubjectDetailView（管理对象详情页）
    ├── CategoryBudgetCard × 3（分类预算卡片，含AI分析）
    ├── Tab 切换：差旅 | 招待 | 团建
    ├── 差旅 Tab
    │   ├── Sub-Tab：出差单 | 酒店 | 机票 | 其他 | 补贴 | 超额
    │   ├── TravelOrderListView（出差单列表）
    │   │   └── TravelOrderCard × N（可展开，含费用管理）
    │   ├── SubTypeSummaryView（酒店/机票/其他汇总）
    │   ├── AllowanceSummaryView（补贴汇总，含已发/未发）
    │   └── ExcessSummaryView（酒店超额汇总）
    ├── 招待/团建 Tab
    │   └── SimpleExpenseListView（费用列表，含搜索）
    └── TravelPolicyModal（差旅政策配置弹窗）
```

### 5.2 状态管理

- 使用 React useState + useCallback 管理本地状态
- 无全局状态管理，各组件独立获取数据
- 数据变更后通过 onRefresh/onDataChanged 回调刷新父组件

### 5.3 第三方依赖

- `xlsx`：Excel 导出（前端生成）
- `apiClient`：封装的 HTTP 客户端
